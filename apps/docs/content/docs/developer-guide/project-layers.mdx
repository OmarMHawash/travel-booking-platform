While the [Core Concepts](/docs/architecture/core-concepts) page explains the theory of Clean Architecture, this guide provides a practical look at what kind of code lives in each layer of a module. Understanding this is crucial for knowing where to add new code.

Let's use the `Hotels` module as an example.

### The Dependency Flow

The most important rule is that **dependencies only point inwards**.

- **API** depends on **Application**.
- **Infrastructure** depends on **Application**.
- **Application** depends on **Domain**.
- **Domain** depends on **nothing**.

This ensures our core business logic (`Domain`) is completely isolated from any technical implementation details, making it highly portable and testable.
